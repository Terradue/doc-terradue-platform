Terradue Open Source Center

With Terradue Open Source Center, we are sharing a set of GitHub software repositories for building Earth Observation applications. You can see there the public repositories that are open to community contributions as of today. They are dynamically ordered by the latest dates of commit.

Terradue is also using Open Source Software managed and curated by 3rd parties, and in some cases Terradue is actively supporting them (e.g. Open Nebula). As a general design principle in our solutions, software APIs (Application Programming Interfaces) are based on open standard specifications (Cloud Computing standards, OGC web services), and we regularly proof check their interoperability.

A simple OSS guide

And finally there's now a GitBook: A Simple Guide to our Open Source Software projects, providing an introduction to the Open Source releases and process adopted by Terradue on GitHub. We regularly work on funded projects that add candidate open source software in this pipeline (managed as GitHub private repositories), with the goal for these projects to go public and gather a community interest.

By joining the ngEO consortium (Next Generation User Services for Earth Observation), in charge of new multi-mission User Services for the European Space Agency, we brought our expertise acquired with the G-POD Web Server, as well as the ESA HMA and the OGC OpenSearch standardization efforts for advanced product search and ordering functions. Within ngEO, we endorsed the responsibility for the development of the "ngEO Web Server", the decision engine orchestrating data discovery & access User Services.
Overall ngEO defines a set of services dealing with product metadata catalogue, browsing images, data access, shop-cart management, authorization & access management. and develops cross-cutting end-user Earth Observation services, integrable on top of different operational contexts, such as the Sentinel Payload Data Ground Segments, the Copernicus Data access Services, or the Multi-mission infrastructure and EarthCARE.

The Grid Processing On Demand (G-POD) facility, a European Space Agency initiated project, provides a user-segment putting Earth Observation data and processors close together. This computing infrastructure operated by ESA ESRIN is serving hundreds of science users and provides cost-effective support for EO data processing operations, and scientific applications development.
With the G-POD Portal server, industrialized by Terradue since mid-2006, we've built a long-standing experience in software maintenance and evolutions, created a core software engineering know-how, and supported a lot of collarobarations with numerous industrial partners contributing to the G-POD applications ecosystem.

A virtual constellation is a set of space and ground segment capabilities, operating together via the cooperation of space agencies. The goal of a constellation is to sustain a critical observation capability, with all the added-value of a collaborative partnership.
With this study, Terradue provided guidance and support to the Centre National d'Etudes Spatiales (CNES), for the compliancy of data collections in terms of data management, discovery and access, in order to improve the overall footprint of the Committee on Earth Observation Satellites (CEOS) virtual constellations within the Global Earth Observing System of Systems (GEOSS). Get an overview (part on "VC Portals"), presented by CNES at the CEOS Strategic Implementation Team SIT-28 meeting in March 2013.

The Open Geospatial Consortium's (OGC) 10th Interoperability Testbed (OWS-10) produced on April 2014 an engineering report on the "Performance of OGC Services in the Cloud" (ref. 14-028r1, access it via this direct download link). This report was contributed by Terradue for an activity in collaboration with CNR-IREA (SBAS processing chain) and benefiting from a grant program of Amazon Web Services Government & Education solutions.
In this work, we demonstrated the performance enhancements for a Cloud deployment of a processing application making use of WPS 1.0 and OpenSearch Geo & Time 1.0 OGC Web Services, for the on-demand production of Ground Deformation Maps. The deployment exploited resources from Terradue (Cloud Controller), AWS (public Cloud), Interoute (public Cloud), and the GEO Supersites Virtual Archive (Cloud Storage of massive ESA SAR data).
