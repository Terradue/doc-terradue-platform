Terradue Open Source Center

With Terradue Open Source Center, we are sharing a set of GitHub software repositories for building Earth Observation applications. You can see there the public repositories that are open to community contributions as of today. They are dynamically ordered by the latest dates of commit.

Terradue is also using Open Source Software managed and curated by 3rd parties, and in some cases Terradue is actively supporting them (e.g. Open Nebula). As a general design principle in our solutions, software APIs (Application Programming Interfaces) are based on open standard specifications (Cloud Computing standards, OGC web services), and we regularly proof check their interoperability.

A simple OSS guide

And finally there's now a GitBook: A Simple Guide to our Open Source Software projects, providing an introduction to the Open Source releases and process adopted by Terradue on GitHub. We regularly work on funded projects that add candidate open source software in this pipeline (managed as GitHub private repositories), with the goal for these projects to go public and gather a community interest.

Cloud Services can be configured to provide an OGC WPS interface to enhance the support of service discovery, description, submission, monitoring and results access. WPS is a specification from the Open Geospatial Consortium (OGC 08-091r6) that proposes a solution for developing web-based geo-processing plug-ins, and for easily sharing algorithms and geo-processing functionality. It defines a standard for publishing, discovery and binding to geospatial processes thus providing a standard way for defining a web services interface for geospatial processing.

For each G-POD service, processing parameters are defined with default or fixed values and different WPS configurations can be defined per processing service. Usually, a WPS interface is fed by input products selected from a machine interaction with a catalogue service interface, matching the correct transmission of the provided parameters (all products matching the catalogue search criteria are selected as input products for the processing).

Over the years, we have served the European Space Agency with numerous software engineering projects. We built a long-standing experience in software maintenance and evolutions, creating a core business know-how in support of the ESA Grid-Processing on Demand (G-POD) system, a flagship facility serving hundreds on scientists since 2006.

Grid Processing on-Demand (G-POD), an ESA-initiated project industrialized by Terradue since mid-2006, provides a user-segment putting EO data and processors closer together. This common shared infrastructure, accessible to science users and operations provides cost-effective support for EO operations, science and application development. The G-POD is a hybrid Grid/Cloud based high-performance and high through-put computing infrastructure promoting the access to Earth Observation data, offering on-line access to products with attached computing infrastructure and tools to generate scientific added-value products.

We regularly deliver our expertise as project lead or consortium member, with some results being published as IPR-free resources (open standards specifications, study reports, communications, ...)
